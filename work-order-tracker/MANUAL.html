Bab 01

Pengenalan Sistem

NOC FMI Web adalah platform internal Fibermedia yang dirancang khusus untuk tim Network Operations Center (NOC). Sistem ini mengintegrasikan berbagai kebutuhan operasional dalam satu tampilan, mulai dari pencatatan gangguan backbone hingga pembuatan laporan otomatis.

Fitur Utama

FiturKeterangan
🔴 Backbone Report NOCManajemen tiket gangguan jaringan backbone
💬 WA ReportGenerate laporan otomatis siap kirim ke WhatsApp
📄 PDF ReportLaporan bulanan/kuartalan/tahunan dalam format PDF
✅ Approval Kode BackboneSistem persetujuan penambahan kode backbone via Telegram
🌐 VLAN DatabaseManajemen database VLAN
👥 Data ClientDatabase client korporat
📋 Work OrdersPengelolaan work order aktivasi dan terminasi
🔗 Integrasi OdooSinkronisasi tiket dengan Odoo Helpdesk
📨 Integrasi TelegramNotifikasi log dan sistem approval via bot Telegram
Bab 02

Login & Akses

Cara Login

  1. Buka browser dan akses URL aplikasi
  2. Masukkan Email dan Password yang telah didaftarkan
  3. Klik tombol Login
⚠️ Akun harus didaftarkan terlebih dahulu oleh Admin atau SUPER_DEV. Tidak ada fitur registrasi mandiri.

Lupa Password

  1. Klik "Lupa Password?" di halaman login
  2. Masukkan email yang terdaftar
  3. Cek email untuk link reset password
  4. Klik link dan buat password baru

Keamanan Akun

  • Gunakan password minimal 6 karakter dengan kombinasi huruf besar, kecil, dan angka
  • Jangan bagikan password ke orang lain
  • Logout setelah selesai menggunakan sistem

Bab 06

Backbone Report NOC

Fitur utama untuk tim NOC dalam mengelola tiket gangguan jaringan backbone secara real-time.

6.1 Melihat Daftar Tiket

  • Buka menu Backbone Report NOC di sidebar
  • Daftar tiket ditampilkan dalam tampilan tabel
  • Gunakan filter di bagian atas untuk menyaring berdasarkan status, periode, atau kode backbone

Status Tiket

StatusWarnaKeterangan
OPEN🔴 MerahTiket baru dibuka, belum ada penanganan
ON PROGRESS🟡 KuningTim sedang menangani gangguan
PENDING🟠 OranyeMenunggu pihak lain / material / informasi
SOLVED🟢 HijauGangguan telah teratasi sepenuhnya
UNSOLVED⚫ AbuTidak dapat diselesaikan
CANCEL⬜ PutihTiket dibatalkan

6.2 Membuat Tiket Baru

  1. Klik tombol "+ Tiket Baru" di pojok kanan atas
  2. Isi form dengan data berikut:
FieldKeterangan
Nomor TiketNomor tiket unik (contoh: HT12345)
Subject TicketJudul/deskripsi singkat gangguan
Tanggal ReportTanggal dan hari kejadian
PriorityMAJOR / CRITICAL / MINOR
Kode BackbonePilih dari daftar Index NOC
Nama LinkNama link yang terdampak
KapasitasKapasitas link (contoh: 10G, 1G)
ProblemJenis problem (DOWN, DEGRADED, dll)
Start DownWaktu mulai gangguan
  1. Klik "Simpan"
Tiket yang dibuat akan otomatis tersinkronisasi ke Odoo Helpdesk jika integrasi sudah dikonfigurasi di profil.

6.3 Update Status Tiket

  1. Klik ikon edit pada baris tiket yang ingin diupdate
  2. Pilih status baru
  3. Isi informasi tambahan sesuai status yang dipilih
  4. Klik "Simpan"
StatusInformasi yang Diisi
ON PROGRESSUpdate timeline / keterangan tindakan
PENDINGAlasan pending / menunggu apa
SOLVEDWaktu selesai dan detail penyelesaian
UNSOLVEDAlasan tidak dapat diselesaikan
CANCELAlasan pembatalan tiket

6.4 Timeline ON PROGRESS

Setiap kali update ke status ON PROGRESS atau PENDING, entri baru akan ditambahkan ke timeline — bukan menimpa yang lama.

Contoh tampilan timeline yang terkumpul:

> Team dalam perjalanan ke lokasi
> Team tiba di lokasi, mulai pengecekan
> Ditemukan kabel putus di titik koordinat X
> Proses penyambungan kabel selesai, monitoring 15 menit

Seluruh riwayat update tersimpan dan ditampilkan di WA Report.

6.5 WA Report

Generate laporan siap kirim ke WhatsApp Group dengan satu klik.

Cara Menggunakan

  1. Klik tombol "WA Report" di toolbar atas
  2. Modal menampilkan preview laporan semua tiket aktif hari ini
  3. Klik "Copy Report" untuk menyalin ke clipboard
  4. Paste ke WhatsApp Group NOC

Contoh Format WA Report

*REPORT BACKBONE PROBLEM | Rabu, 29 April 2026*

===================================
1. HT12345 - MAJOR - BACKBONE - MR <> SAMALI V DIRECT 10G - DOWN [Selasa 29 April 2026]

Problem    : DOWN
Impact     :
- SAMALI <> APJII V BUNCIT (10G)
Status     : ON PROGRESS

> Team dalam perjalanan ke lokasi
> Team tiba di lokasi, mulai pengecekan
Laporan hanya menampilkan tiket dengan status aktif (OPEN, ON PROGRESS, PENDING).

6.6 PDF Report

Generate laporan rekap dalam format PDF untuk periode tertentu.

Cara Menggunakan

  1. Klik tombol "PDF" di toolbar atas
  2. Pilih tipe laporan:
    • Bulanan — Pilih bulan dan tahun
    • Kuartalan — Pilih kuartal dan tahun
    • Tahunan — Pilih tahun
  3. Klik "Generate PDF"

Isi PDF Report

  • Rekap jumlah tiket per status
  • Statistik SLA dan MTTR rata-rata
  • Breakdown per jenis problem
  • Breakdown per kode backbone
  • Tabel detail semua tiket dalam periode
Item dengan jumlah 0 tetap ditampilkan (dengan tampilan redup) sebagai referensi kelengkapan data.

6.7 Kode Backbone (Index NOC)

Daftar kode backbone yang terdaftar sebagai referensi saat membuat tiket.

Menambah Kode Backbone

RoleProses
SUPER_DEV / ADMINLangsung masuk ke Index NOC tanpa perlu approval
NOCMasuk ke antrian approval, menunggu persetujuan Admin

6.8 Sistem Approval Kode Backbone

Untuk menjaga kualitas data, penambahan kode backbone oleh role NOC harus disetujui Admin/SUPER_DEV terlebih dahulu.

NOC request kode baru ↓ Masuk ke backbone_pending ↓ Bot Telegram kirim DM notifikasi ke semua Admin ↓ Admin klik [✅ Setujui] atau [❌ Tolak] di Telegram ↓ Disetujui → Kode masuk ke Index NOC Ditolak → Request dihapus

Langkah untuk User NOC

  1. Klik tombol "+ Kode Backbone"
  2. Isi Kode dan Nama Backbone
  3. Klik "Request"
  4. Notifikasi otomatis terkirim ke Admin via Telegram
  5. Tunggu persetujuan — kode muncul di Index NOC setelah disetujui

Langkah untuk Admin (via Web)

  • Badge ⏳ Acc Kode muncul di toolbar saat ada request pending
  • Klik badge → lihat daftar request → Setujui atau Tolak

Langkah untuk Admin (via Telegram)

  • Bot mengirim DM berisi detail request
  • Klik ✅ Setujui atau ❌ Tolak langsung dari chat Telegram
  • Pesan otomatis terupdate dengan status hasil keputusan

Bab 17

Profil & Pengaturan Akun

Informasi Pribadi

  1. Klik nama/avatar di pojok kiri bawah sidebar → Profil
  2. Update Nama Lengkap dan Foto Profil (maks. 2 MB)
  3. Klik "Simpan Perubahan"

Ganti Password

  1. Di halaman Profil, scroll ke bagian "Keamanan Akun"
  2. Isi Password Baru dan Ulangi Password
  3. Klik "Ganti Password"

Integrasi Odoo Helpdesk

Agar tiket otomatis tersinkron ke Odoo, setiap user perlu mengatur API Key-nya sendiri:

  1. Di halaman Profil, scroll ke "Integrasi Odoo Helpdesk"
  2. Cara mendapatkan API Key Odoo:
    • Login ke Odoo dengan email yang sama
    • Klik foto profil → Preferences
    • Buka tab Account Security
    • Klik New API Key → beri nama → Generate
    • Salin API Key (hanya ditampilkan sekali!)
  3. Paste API Key ke field "Odoo API Key"
  4. Klik "Simpan & Verifikasi"
  5. Status berubah menjadi "Terhubung" jika berhasil
⚠️ API Key bersifat rahasia. Tiket akan tercatat atas nama akun Odoo kamu.
Bab 18

Manage Users

Halaman khusus Admin dan SUPER_DEV untuk mengelola akun user.

Mengubah Role User

  1. Buka menu Manage Users
  2. Klik user yang ingin diubah
  3. Pilih role baru dari dropdown
  4. Klik "Simpan"

Override Permission

Untuk memberikan akses khusus di luar role default:

  1. Klik user yang ingin diatur
  2. Di bagian "Permission Override", aktifkan/nonaktifkan permission spesifik
  3. Override menimpa hak akses role default untuk user tersebut

Mengundang User Baru

  1. Klik "+ Tambah User"
  2. Masukkan email user baru
  3. Pilih role yang sesuai
  4. Klik "Kirim Undangan"

User akan mendapat email undangan untuk mengatur password pertama kali.

Bab 19

Integrasi Telegram

Sistem memiliki 2 Bot Telegram untuk mendukung operasional NOC.

Bot 1 — Notifikasi Log

Mengirim notifikasi aktivitas penting ke Group Telegram NOC secara otomatis.

Contoh Notifikasi yang Dikirim

  • Tiket baru dibuat
  • Status tiket diubah (OPEN → ON PROGRESS, dsb)
  • Tiket diselesaikan

Bot 2 — Sistem Approval Backbone

Mengirim DM (Direct Message) ke Admin saat ada request kode backbone baru dari user NOC.

Cara Kerja

  1. User NOC request kode backbone baru
  2. Bot mengirim DM ke semua Admin dengan tombol inline ✅ Setujui / ❌ Tolak
  3. Admin klik tombol langsung dari Telegram
  4. Pesan di-update otomatis dengan status hasil keputusan dan nama approver

Syarat Penggunaan Bot

  • Admin harus pernah klik START ke Bot 2 terlebih dahulu
  • Telegram User ID Admin harus terdaftar di sistem

Cara Mendapatkan Telegram User ID

  1. Buka Telegram
  2. Cari bot @userinfobot
  3. Klik Start
  4. Bot membalas dengan User ID kamu
  5. Berikan User ID ke Admin sistem untuk didaftarkan
Bab 20

Integrasi Odoo Helpdesk

Sistem terintegrasi dengan Odoo Helpdesk untuk sinkronisasi tiket secara otomatis.

Cara Kerja

Setiap tiket yang dibuat atau diupdate di NOC Web akan otomatis:

  • Membuat tiket baru di Odoo (jika belum ada)
  • Menambahkan log note saat status berubah
  • Mencatat nama teknisi yang melakukan update

Status Sinkronisasi

StatusKeterangan
✅ TerhubungIntegrasi berjalan normal
⚠️ Belum DikonfigurasiUser belum set API Key di profil
❌ GagalPeriksa API Key atau koneksi ke Odoo
